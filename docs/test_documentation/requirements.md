# Функциональные требования по модулям

## 1. Модуль: Регистрация и Авторизация

**Требование 1.1:** Система должна предоставлять возможность регистрации нового пользователя с помощью email и пароля.

**Требование 1.2:** Система должна предоставлять возможность регистрации/авторизации через социальные сети (VK, Одноклассники, Яндекс, Mail.ru и т.д.).

**Требование 1.3:** При регистрации должна проводиться валидация вводимых данных (корректность email, минимальная сложность пароля).

**Требование 1.4:** Система должна предоставлять возможность авторизации существующего пользователя по логину (email/телефон) и паролю.

**Требование 1.5:** Система должна предоставлять функцию "Забыли пароль?" для его восстановления.

## 2. Модуль: Поиск и Каталог Товаров

**Требование 2.1:** Поисковая строка на главной странице должна возвращать релевантные результаты по введенному запросу.

**Требование 2.2:** Система должна предоставлять возможность фильтрации результатов поиска (по цене, рейтингу, бесплатной доставке, стране-отправителю и т.д.).

**Требование 2.3:** Система должна предоставлять возможность сортировки результатов поиска (по популярности, новизне, рейтингу, цене).

**Требование 2.4:** Пользователь может просматривать товары по категориям и подкатегориям через главное меню.

## 3. Модуль: Карточка Товара

**Требование 3.1:** На странице товара должна отображаться вся ключевая информация: галерея фото, название, цена, выбор вариантов (цвет, размер), рейтинг, количество отзывов, кнопка "В корзину" и "Купить сейчас".

**Требование 3.2:** Должна быть возможность просмотра увеличенных изображений товара.

**Требование 3.3:** Должна быть возможность просмотра отзывов и рейтингов от других покупателей.

**Требование 3.4:** Должна быть возможность выбора характеристик товара (если применимо).

**Требование 3.5:** Должна отображаться информация о доставке (стоимость, сроки, способы).

## 4. Модуль: Корзина

**Требование 4.1:** Система должна добавлять выбранный товар в корзину при нажатии кнопки "В корзину".

**Требование 4.2:** В корзине должна отображаться сводная информация: список товаров, их количество, цена за единицу, общая стоимость.

**Требование 4.3:** Пользователь должен иметь возможность изменять количество товара или удалять его из корзины.

**Требование 4.4:** Должна корректно рассчитываться итоговая сумма с учетом скидок и стоимости доставки.

## 5. Модуль: Оформление Заказа

**Требование 5.1:** Процесс оформления заказа должен быть доступен только для авторизованных пользователей.

**Требование 5.2:** Система должна запрашивать/предлагать выбор адреса доставки.

**Требование 5.3:** Система должна предлагать на выбор доступные способы оплаты (карта, электронные кошельки и т.д.).

**Требование 5.4:** После подтверждения заказа система должна перенаправлять пользователя на страницу успешного оформления и присваивать заказу уникальный номер.

## 6. Модуль: Личный Кабинет и "Мои Заказы"

**Требование 6.1:** Пользователь должен иметь возможность просматривать историю своих заказов.

**Требование 6.2:** Для каждого заказа должен отображаться его статус ("Обрабатывается", "Отправлен", "Доставлен").

**Требование 6.3:** Пользователь должен иметь возможность отслеживать посылку по трек-номеру.

**Требование 6.4:** Пользователь должен иметь возможность редактировать свои персональные данные и адреса доставки.

## 7. Модуль: Избранное

**Требование 7.1:** Пользователь должен иметь возможность добавлять товары в "Избранное" (нажимая на иконку сердца).

**Требование 7.2:** Список избранных товаров должен сохраняться в личном кабинете.
